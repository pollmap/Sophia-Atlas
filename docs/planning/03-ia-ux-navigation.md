## 1. IA 설계 철학

### 1.1 두 가지 사용 모드

Sophia Atlas 사용자는 크게 두 가지 모드로 사이트를 이용한다:

```
┌─────────────────────────────────────────────────┐
│                                                   │
│  목적형 (Purposeful)          탐험형 (Exploratory) │
│  "소크라테스를 찾고 싶다"       "뭔가 재밌는 걸 보고 싶다" │
│                                                   │
│  검색 → 결과 → 상세 페이지     홈 → 그래프 → 클릭 → │
│                                발견 → 관련인물 →    │
│  빠른 도달이 핵심               세렌디피티가 핵심    │
│                                                   │
│  수단: 검색, 필터, 목록        수단: 그래프, 타임라인, │
│                                지도, 랜덤 추천       │
└─────────────────────────────────────────────────┘
```

### 1.2 3클릭 규칙

어떤 인물이든 **3클릭 이내**에 도달 가능해야 한다:

- **경로 1**: 홈 → 검색 → 결과 클릭 (2클릭)
- **경로 2**: 홈 → 카테고리 허브 → 인물 클릭 (2클릭)
- **경로 3**: 홈 → 인드라망 그래프 → 노드 클릭 (2클릭)
- **경로 4**: 헤더 → 드롭다운 인물 → 인물 클릭 (2클릭)

### 1.3 정보 계층

```
L0: 홈 (전체 개요)
├── L1: 도메인 허브 (철학, 종교, 과학, 역사, 문화)
│   ├── L2: 도메인 뷰 (타임라인, 그래프, 지도, 비교)
│   └── L2: 인물 상세 (/philosophy/[id], /religion/[id])
├── L1: 통합 뷰 (인물목록, 엔터티목록, 인드라망)
│   └── L2: 개별 상세 (/persons/[id], /entities/[id])
└── L1: 도구 (검색, 학습, About)
```

## 2. 사이트 구조

### 2.1 전체 사이트맵

```
/                                    # 홈 대시보드
├── /philosophy/                     # 철학 허브
│   ├── /philosophy/timeline         # 철학 타임라인
│   ├── /philosophy/graph            # 영향 관계 그래프
│   ├── /philosophy/questions        # 철학적 질문 탐색
│   └── /philosophy/[id]             # 개별 철학자 페이지
│
├── /religion/                       # 종교 (허브 = 지도)
│   ├── /religion/map                # 세계 종교 지도
│   ├── /religion/tree               # 종교 분파 트리
│   ├── /religion/compare            # 종교 비교 매트릭스
│   └── /religion/[id]               # 개별 종교 페이지
│
├── /science/                        # 과학 허브
│   ├── /science/timeline            # 과학 타임라인
│   ├── /science/fields              # 분야별 탐색
│   └── /science/[id]                # (미래) 개별 과학자
│
├── /history/                        # 역사 허브
│   ├── /history/timeline            # 역사 타임라인
│   ├── /history/civilizations       # 문명별 탐색
│   └── /history/[id]                # (미래) 개별 역사인물
│
├── /culture/                        # 문화 허브
│   ├── /culture/movements           # 문화 운동/사조
│   └── /culture/[id]                # (미래) 개별 문화인물
│
├── /persons/                        # 통합 인물 목록
│   └── /persons/[id]                # 통합 인물 상세
│
├── /entities/                       # 엔터티 목록
│   └── /entities/[id]               # 엔터티 상세
│
├── /connections                     # 인드라망 시각화
├── /search                          # 통합 검색
├── /learn                           # 학습 경로
└── /about                           # 프로젝트 소개
```

### 2.2 페이지 유형 분류

| 유형 | 페이지 | 목적 | 데이터 전략 |
|------|--------|------|------------|
| **Hub** | /, /philosophy, /science, /history, /culture | 도메인 진입점, 개요 | SSG, 경량 데이터 |
| **List** | /persons, /entities | 필터링 가능한 전체 목록 | SSG + CSR 필터 |
| **Detail** | /persons/[id], /entities/[id] | 개별 항목 상세 | SSG (generateStaticParams) |
| **Visualization** | /connections, /*/timeline, /*/graph | 인터랙티브 시각화 | SSG + CSR (D3/Leaflet) |
| **Tool** | /search, /religion/compare | 사용자 도구 | CSR 중심 |
| **Content** | /learn, /about | 정적 콘텐츠 | SSG |

### 2.3 URL 설계 원칙

```
규칙 1: 영문 kebab-case
  ✅ /philosophy/timeline
  ❌ /philosophy/타임라인

규칙 2: 계층 구조 반영
  ✅ /religion/compare
  ❌ /compare-religions

규칙 3: 동적 라우트는 [id] 패턴
  ✅ /persons/socrates
  ❌ /persons?id=socrates

규칙 4: 필터/정렬은 쿼리 파라미터
  ✅ /persons?category=philosopher&era=ancient
  ❌ /persons/philosopher/ancient

규칙 5: 뷰 모드는 쿼리 파라미터
  ✅ /persons?view=grid
  ❌ /persons/grid-view
```

## 4. 사용자 여정

### 4.1 페르소나

| 페르소나 | 특징 | 주요 행동 |
|----------|------|-----------|
| **대학생 민지** | 철학 전공 3학년, 과제 때문에 방문 | 검색 → 상세 → 관계 → 인용 |
| **교양인 준호** | 40대 직장인, 주말 저녁 호기심 탐색 | 홈 → 그래프 → 클릭 탐험 → 공유 |
| **연구자 김교수** | 비교종교학 교수, 데이터 확인 | 검색 → 비교 매트릭스 → 상세 검증 |
| **교사 이선생** | 고등학교 윤리 교사, 수업 자료 | 타임라인 → 인물 카드 → 학습 경로 |
| **캐주얼 방문자** | SNS에서 명언 공유 보고 유입 | 인물 상세 → 명언 → 관련 인물 |

### 4.2 시나리오별 플로우

#### 시나리오 1: "칸트에 대해 알고 싶다" (목적형)

```
[검색바 클릭] → [칸" 입력] → [자동완성: 칸트] → [클릭]
     │
     ↓
[/persons/kant 페이지]
     │
     ├── 요약 읽기 (5초)
     ├── "상세 설명" 펼치기 (30초)
     ├── "주요 저작" 확인
     ├── "관계 미니그래프" 확인
     │     └── [흄] 클릭 → /persons/hume
     └── "명언" 공유
```

#### 시나리오 2: "고대 그리스 철학을 둘러보고 싶다" (탐험형)

```
[홈] → [철학 카드 클릭] → [/philosophy]
     │
     ↓
[/philosophy/timeline]
     ├── 고대 시대 영역으로 스크롤
     ├── [소크라테스 노드] 호버 → 카드 팝업
     ├── [플라톤 노드] 클릭 → /philosophy/plato
     │     └── 관련 인물에서 [아리스토텔레스] 클릭
     └── 필터: "소크라테스 이전" 선택
           └── [탈레스] [헤라클레이토스] [파르메니데스] 발견
```

#### 시나리오 3: "프랑스 혁명에 영향을 준 사상가" (관계 추적형)

```
[검색: "프랑스 혁명"] → [/entities/french-revolution]
     │
     ├── 관련 인물 섹션 확인:
     │   [루소] [볼테르] [몽테스키외] [칸트]
     │
     ├── [루소] 클릭 → /persons/rousseau
     │     └── 영향 관계: 루소 → 프랑스 혁명 (caused)
     │     └── 루소의 영향 받은 인물: [로크] [홉스]
     │
     └── [인드라망에서 보기] 클릭 → /connections?focus=french-revolution
           └── 그래프에서 프랑스 혁명 중심 네트워크 시각화
```

## 6. 반응형 설계

### 6.1 브레이크포인트

| 이름 | 범위 | 컬럼 | 사이드바 | 네비게이션 |
|------|------|------|---------|-----------|
| **Mobile** | < 640px | 1 | 오버레이 | 햄버거 |
| **Tablet** | 640-1024px | 2 | 오버레이 | 축약 메뉴 |
| **Desktop** | 1024-1440px | 3 | 고정 | 전체 메뉴 |
| **Wide** | > 1440px | 4 | 고정 | 전체 메뉴 |

### 6.2 레이아웃 전환

```
Mobile (1col):
┌──────────┐
│ Card 1   │
├──────────┤
│ Card 2   │
├──────────┤
│ Card 3   │
└──────────┘

Tablet (2col):
┌──────┬──────┐
│ C1   │ C2   │
├──────┼──────┤
│ C3   │ C4   │
└──────┴──────┘

Desktop (3col):
┌────┬────┬────┐
│ C1 │ C2 │ C3 │
├────┼────┼────┤
│ C4 │ C5 │ C6 │
└────┴────┴────┘
```

### 6.3 시각화 반응형

| 시각화 | Mobile | Tablet | Desktop |
|--------|--------|--------|---------|
| 그래프 | 전체폭, 터치 제스처 | 전체폭 | 사이드패널 + 그래프 |
| 타임라인 | 세로 스크롤 | 가로 스크롤 | 가로 스크롤 + 줌 |
| 지도 | 전체폭, 하단 리스트 | 전체폭 + 사이드 리스트 | 사이드패널 + 지도 |
| 비교 매트릭스 | 가로 스크롤 | 가로 스크롤 | 전체 표시 |

## 8. 접근성 & SEO

### 8.1 접근성 (A11y)

- **키보드 네비게이션**: 모든 인터랙티브 요소 Tab 접근 가능
- **스크린 리더**: 시맨틱 HTML + ARIA 라벨
- **색상 대비**: WCAG AA (4.5:1 텍스트, 3:1 UI)
- **포커스 관리**: 모달/드롭다운 포커스 트랩
- **대체 텍스트**: 모든 이미지/아이콘에 alt/aria-label
